Writing Blog with Hexo

Posted on 2019-02-22

Record some tips to establish and write an article with hexo.

Establish an article

Enter from the command line (title means title of the article):

Then hexo will create a new file in source/_post with is named: title.md. It includes the contents of our article, just edit it with any convenient editor.

Some settings in hexo:
1.Turn the post_asset_folder in _config.yml into true
2.Install a package:npm install hexo-asset-image --save
3.Then a folder will be created in source/_post with the same name as the article. Just put our figures into the folder and use them with the Markdown grammar.

Template

When we create our articles with hexo new "title" , hexo will initialize the new file based on /scaffolds/post.md. It means that post.md is the template of the new article. As a result, we can modify it for our own writing habits.

Header

Header includes some discription of the article.

Front page

Contents of articles will be displayed in the front page of our blog (All the contents are displayed in default). When the article is long, it is necessary to reduce the display. We can write <!--more--> before the contents which is unnecessary to be shown.
